File Structure

The template contains 16 html files. You will find them in the root directory of the download package.

Css

The template comes with a number of CSS files, many of which you will likely never touch. There are 2 files that you will most likely edit since you are going to apply some degree of customization to your site.

style.css

This is the main stylesheet for the template and includes all styles relating to the appearance of the elements you see inside each page. This stylesheets has been commented for ease of use and all style rules are grouped under particular headings such as /* Typography */

color-red.css

This is the main color scheme for Chair. Template. However if you would prefer a diffrent one you will need to include the appropriate CSS file in the head of your document. For example, if you wanted to use the 'Cyan' colour scheme you would include the following:

<link href="css/color-cyan.css" rel="stylesheet" type="text/css" media="all"/>
Creating your own colour scheme with SASS

As Chair. comes with scss file for color scheme, you can easily change the colour scheme in color.less The colour variable is as follows:

$primary-color

Edit this value inside the color.scss file and compile to generate your own colour scheme then include it in your html documents as showen above

Javascript Files

Chair. comes with a number of JS files, many of which you will never edit. If you don't know your way around javascript please don't edit them, these files will break parts of your site.

scripts.js

This is the main scripts file for the template, that includes all functions specific to the theme such as control of the slider and parallax declarations among other things. You will see that the file is structured as

$(document).ready(function(){});

followed by some other helper functions.

Grid System

The template is based on Bootstrap v3.3.5, 12 column grid system. Is flexible and easy to understand. For more info go here.

Basic Introduction

Grid systems are used for creating page layouts through a series of rows and columns that house your content. Here's how the Bootstrap grid system works:

Rows must be placed within a .container (fixed-width) or .container-fluid (full-width) for proper alignment and padding.

Use rows to create horizontal groups of columns.

Content should be placed within columns, and only columns may be immediate children of rows.

Predefined grid classes like .row and .col-xs-4 are available for quickly making grid layouts. Less mixins can also be used for more semantic layouts.

Columns create gutters (gaps between column content) via padding. That padding is offset in rows for the first and last column via negative margin on .rows.

The negative margin is why the examples below are outdented. It's so that content within grid columns is lined up with non-grid content.

Grid columns are created by specifying the number of twelve available columns you wish to span. For example, three equal columns would use three .col-xs-4.

If more than 12 columns are placed within a single row, each group of extra columns will, as one unit, wrap onto a new line.

Grid classes apply to devices with screen widths greater than or equal to the breakpoint sizes, and override grid classes targeted at smaller devices. Therefore, applying any .col-md- class to an element will not only affect its styling on medium devices but also on large devices if a .col-lg- class is not present.

Parallax

This effect is disabled for touch devices to improve performance.

Enabling the parallax effect requires you to add the class '.parallax-background' to the '.background-image-holder' element like so:

<section> 
    <div class="background-image-holder parallax-background"> 
        <img alt="Background Image" class="background-image" src="images/image.jpg" /> 
    </div> 
</section>

Lightbox

Chair. Template makes use of the lightbox gallery javascript plugin and is simple to setup on any image element by observing the following conventions. The image must be wrapped in an 'a' tag and contain a href that points to the large image (the one you want to show when the user clicks on the thumbnail). Inside the 'a' you need an 'img' element whose 'src' value points to the thumbnail image. 

<a href="images/large-image.jpg" class="lightbox"> 
    <img src="images/thumbnail.jpg" alt=""> 
</a>

Also you need to call the script for lightbox in that page, if it's not already there place the following line after footer and before body tag ends:

<script src="js/popup.min.js"></script>

Icons

Chair. makes use of two awesome icon packs. The first one is Font Awesome and the second Linea.Using these icon packs is a simple matter of knowing the correct class to use.

Font Icon Usage:

<i class="fa fa-NAME"></i> Example: <i class="fa fa-paper-plane"></i>

Linea Font usage:

<span data-icon="v" class="icon"></span>

Slider

Chair. uses Slick Slider plugin as its primary slider manager. You can view all available options for the plugin at their website

The basic structure implemented in Chair. Template is as follows:

<header class="fullscreen-element">
    <div class="hero-slider">
        <div class="slide" style="background: url('images/image.jpg') 50% 0%;">
            ...YOUR CONTENT HERE...
        </div>
        <div class="slide" style="background: url('images/image.jpg') 50% 0%;">
            ...YOUR CONTENT HERE...
        </div>
    </div>
</header>

Map

Chair. uses Google maps plugin. If you want to add a map to any page you have to first call the Google maps API after the footer and before the ending body tag like this:

<script src="https://maps.googleapis.com/maps/api/js?v=3.exp"></script>

Then add a div with the id #map, data-lat="" for latitude and data-long="" for longitude. Like this:

<div id="map" data-lat="40.7033127" data-long="-73.979681"></div>

How to update from 1.0 to 1.1

In version 1.1 we replaced skrollr.js prallax plugin with stellar.js. Skrollr was causing an error on mobile devices and we couldn't scroll down the page. We tried to fix that but then it caused another error, the page height was not acurate and it caused a hudge blank space at the bottom of each page that was using parallax. So we decided to move to a better solution: stellar.js.

If you download the package on TF now the site works just fine without any errors, however if you want to update the old version please do the folowing (please contact us throught our profile page if you can't do it and we can do it for you):

1. Download the package from Themeforest downloads page

2. Extract the package and inside you will find Chair folder. Open it and inside look for js folder. Inside the js folder search for stellar.min.js and copy it to your js folder on the old version.

3. On the old version open js folder, find and delete skrollr.min.js.

4. Replace 

<script type="text/javascript" src="js/skrollr.min.js"></script>

with:

<script type="text/javascript" src="js/stellar.min.js"></script>

on all HTML documents (should be at the very bottom of the page)

5.In the js/scripts.js on line 40 - 43 you will find and delete:

skrollr.init({
    forceHeight: false
});

Instead place this code:

$.stellar({
    horizontalScrolling: false,
    verticalScrolling: true,
    horizontalOffset: 0,
    verticalOffset: 0,
    responsive: true,
    parallaxBackgrounds: true
});

6. Add 

data-stellar-background-ratio="0.9"
to every background you want to use parallax on html like:

<div class="background-image-holder parallax-background overlay" data-stellar-background-ratio="0.9"> 
    <img class="background-image" alt="Background Image" src="images/1600x900.png"> 
</div>

You have to do this for every element that has a class of parallax-background
